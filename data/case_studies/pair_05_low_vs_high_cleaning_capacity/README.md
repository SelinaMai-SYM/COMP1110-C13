# Pair 05: Low vs High Cleaning Capacity

This pair changes only the cleaning and service capacity while keeping arrivals, table layout, queue structure, reservation holding, and seating policy fixed. Version A models a tighter cleaning bottleneck, while Version B allows more seating and reset actions at the same event time. Compare `average_wait_time`, `max_queue_length`, `groups_served`, `table_utilization_overall`, and `service_level_within_30_min` to show that throughput can be constrained by operations as well as table supply.
