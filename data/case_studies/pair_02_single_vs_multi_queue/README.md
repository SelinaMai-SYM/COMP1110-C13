# Pair 02: Single Queue vs Multi Queue

This pair changes only the queue structure while keeping the arrival stream, table layout, reservation holding, and seating/service policy fixed. Version A places every group in one common queue, whereas Version B uses size-based queues. Compare `average_wait_time`, `max_queue_length`, `groups_served`, `service_level_within_15_min`, and `average_table_fit_efficiency` to study the trade-off between operational simplicity and matching efficiency.
